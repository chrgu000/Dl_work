使用说明

1. 系统：WindowsXP, Windows 7, Windows 8, Windows 10。

2. 浏览器：新版订单系统未做IE系列的兼容性测试，为了避免出现异常，强烈建议使用FireFox火狐浏览器( 点我下载 )或者Chrome谷歌浏览器。

3. 分辨率：推荐使用1280*768以上的分辨率，为了获得最佳体验请使用1920*1080的分辨率。

4. 网络：必须。

客户代码

客户代码列表

用户自定义代码后，可以新增普通订单页面直接输入定义的代码，快速选择出产品。

自定义代码规则：

原则上您可以使用任意字母、数字、符号、日文、俄文、罗马文等来定义您自己的产品编码，当然有以下几条规则需要注意：

自定义代码长度不能超过50个字符。

在使用标点符号的时候请注意全角和半角会被系统认定为完全不同的两个符号。

自定义代码不可重复，而且一个产品只能对应一个自定义代码。

自定义代码不区分字母大小写，所有大写字母都将自动转换成为小写。

自定义代码前后的空格符将会被自动去掉，但是代码内部的空格将会被保留。

如何丢掉您的鼠标

请仔细阅读以下的操作指南,这将会大幅提升您的产品录入速度

输入客户自定义代码或数量后可直接按回车键(Enter),系统将会获取客户代码所对应的产品。

光标所在的输入框会有背景色提示，按下TAB键可以将光标移到下一个输入框，按下SHIFT+TAB键光标移到前一个输入框。

数量不是必须输入项，你可以添加完产品后再按照以前的方式逐个录入数量。

成功获取产品并自动添加到下方列表后,系统会清除这四个输入框中的内容，光标会重新回到"客户代码"输入框中，你可以继续录入代码。

获取产品发生错误时，系统会给出提示信息，并给出相应的提示，光标会回到"客户代码"输入框中，你可以继续录入代码。

三个数量输入框均不能输入非数字类型的字符，若输入其他字符，输入框会自动清空。

基本数量输入框可输入小数，其余两个输入框只能输入整数。

凑整选项框说明

当获取的产品为管材时，若基本数量不为一根管材的整倍数时，系统会添加该产品到下方列表，基本数量会根据选项框的选择来重新计算。

举个例子：

如果您的产品为3.8米/根的管子，您在基本数量处输入的是12，计算下来管子为三根多，不到四根

当选择项为“添加凑整”时，基本数量会被重新计算为15.2米，即四根
当选择项为“减少凑整”时，基本数量会被重新计算为11.4米，即三根
当选择项为“重置为0”时，基本数量会被重新计算为0米

当获取的产品为管件时，若基本数量为小数时，系统会添加该产品到下方列表，但是基本数量会设置为0。

举个例子：

如果您的产品为管件，您在基本数量处输入的是5.6，

当选择项为“添加凑整”时，基本数量会被重新计算为6
当选择项为“减少凑整”时，基本数量会被重新计算为5
当选择项为“重置为0”时，基本数量会被重新计算为0米

产品需求

在下产品需求订单之前,请先仔细阅读订单规则及流程


产品需求订单规则

本次产品需求订单只针对常规产品,特殊产品还是在特殊订单里提交;

产品需求订单不等于缺货信息收集,它在审核通过后会自动转换成为正式订单,能保证提到该订单所下的产品数量;

一张产品需求订单只能下一样产品,并且一车必须全部提完,不能分多次提货.

若需求订单一次未提完,则该订单将会被关闭,并且会扣减相应的信用分数,具体规则参看后面信用规则.

产品需求订单流程图

1、【待审核】 提交产品需求订单，只有在此状态下，下单账号才能关闭订单;

2、【审核中】 公司相关部门对需求订单进行审核，若审核未通过，则会关闭该需求订单，并会推送微信消息，告知关闭原因，审核通过则转为正式订单，开始计算信用分数;

3、【生产中】 审核通过的订单，生产部门则开始生产该需求订单所需的产品;

4、【可提货】 产品已生产完成，可以到厂提货，请务必按照规定的时限到厂提货，逾期未提货的订单也将被自动关闭，并会被扣减信用分数，在该状态下客户可更改提货方式为自提的车辆和司机信息，改为真实准确的车辆和司机信息;

5、【已完成】 需求订单已完成提货，若有未提完的部分，订单也会自动关闭;

信用列表

为了更加有序高效的处理好客户的每一个需求，同时也需要客户在提出需求的时候 能够谨慎对待， 使我们能最大程度的协调好各方资源来满足客户的实际产品需求， 因此我们需要建立一个用于客户产品需求的信用规则， 在此基础上双方密切合作完成每一个产品需求。

一、每个客户初始信用分为100分。

二、若信用分低于或等于规定信用分数，则不能进行需求订单下单操作。

三、需求订单正式生效(审核通过)后才开始计算信用。

四、信用计算规则，只要实际提货数量小于订单数量，则信用分数扣减10分。
